# AB/n 实验样本量计算器

一个用于估算 AB 实验所需样本量的在线计算器。适合在实验开始前，快速判断需要多少样本、预计跑多久，以及多组实验时应该如何做多重比较校正。

在线访问：[https://lipeakter9978-hash.github.io/ab-test-calculator/](https://lipeakter9978-hash.github.io/ab-test-calculator/)

## 功能

- 支持填写基准转化率、期望提升幅度、置信水平、统计功效和每日流量
- 支持相对提升和绝对提升两种口径
- 支持 2–10 组 AB/n 实验
- 支持设置多个对照组和多个实验组
- 自动计算每组样本量、总样本量、合并对照样本量和预计实验周期
- 使用 Bonferroni 方法控制多实验组比较带来的整体误报率

## 计算口径

这个计算器基于标准的两比例 Z 检验，用来估算转化率类指标的最小样本量。

多组实验的处理方式：

1. 默认各组等流量分配。
2. 多个对照组会在正式效果计算时合并。
3. 每个实验组分别与合并后的对照组比较。
4. 多个实验组会带来多次比较，因此使用 Bonferroni 校正整体置信水平。

需要注意：多个对照组可以帮助做 A/A 一致性检查，但不能从根本上消除人群差异。如果用户结构差异明显，应优先使用分层随机，例如按地域、端、新老用户等关键特征先分层，再在层内随机分组。

## 参数说明

| 参数 | 含义 |
| --- | --- |
| 基准转化率 | 当前对照版本的转化率 |
| 期望提升幅度 | 希望实验能识别出的最小效果 |
| 相对提升 | 在当前转化率基础上按比例提升，例如 8% 提升 10% 后为 8.8% |
| 绝对提升 | 直接提升多少个百分点，例如 8% 提升 1% 后为 9% |
| 置信水平 | 控制误报率，常用 95% |
| 统计功效 | 如果真实提升存在，实验能识别出来的概率，常用 80% |
| 实验总组数 N | 对照组和实验组数量之和 |
| 对照组数量 | 用于基准比较和 A/A 检查的组数 |
| 每日可用流量 | 用于估算实验需要运行多少天 |
